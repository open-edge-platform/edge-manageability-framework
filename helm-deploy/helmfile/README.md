# Helmfile-based Deployment

Declarative replacement for `helm-deploy-orch.sh` using [helmfile](https://github.com/helmfile/helmfile).

## Prerequisites

```bash
# Install helmfile (v1.1+)
brew install helmfile   # macOS
# or download from https://github.com/helmfile/helmfile/releases

# Install required plugins
helmfile init
```

## Usage

All commands run from the `helmfile/` directory:

```bash
cd helm-deploy/helmfile
```

### Deploy all apps

```bash
# vPro profile
helmfile -e onprem-vpro apply

# EIM profile
helmfile -e onprem-eim apply
```

### Deploy specific app(s)

```bash
# Single app
helmfile -e onprem-vpro -l name=vault apply

# Multiple apps
helmfile -e onprem-vpro -l name=traefik -l name=traefik-pre apply
```

### Preview changes (diff)

```bash
helmfile -e onprem-vpro diff
```

### List releases

```bash
helmfile -e onprem-vpro list
```

### Destroy (uninstall all)

```bash
helmfile -e onprem-vpro destroy
```

### Template only (dry-run)

```bash
helmfile -e onprem-vpro template
```

### Show deployment order (DAG)

```bash
helmfile -e onprem-vpro show-dag
```

## How It Works

1. **Environments** (`environments/onprem-*.yaml`) define which apps are enabled via `apps.<name>: true/false`
2. **Releases** in `helmfile.yaml.gotmpl` use `installed: {{ .Values | get "apps.<name>" false }}` to conditionally deploy
3. **DAG ordering** via `needs:` ensures correct install order (CRDs ‚Üí operators ‚Üí databases ‚Üí apps)
4. **Hooks** run pre-install cleanup (vault secret removal, PG table truncation) via `hooks/pre-install.sh`
5. **Values files** are the same `values_overwrite/` files generated by `generate-values.sh`

## Compared to helm-deploy-orch.sh

| Feature | helm-deploy-orch.sh | helmfile |
|---|---|---|
| Install ordering | Sequential from app-list file | DAG-based with `needs:` |
| Diff before apply | No | `helmfile diff` / `helmfile apply` |
| Rollback | Manual | `helm rollback` per release |
| Concurrency | None (sequential) | Parallel within DAG groups |
| State tracking | None | Helm release history |
| Profile selection | `ORCH_INSTALLER_PROFILE` env | `helmfile -e <env>` |
| Selective deploy | `./helm-deploy-orch.sh install app1 app2` | `helmfile -l name=app1 apply` |

## Adding a New App

1. Add a release block in `helmfile.yaml.gotmpl`
2. Set `installed: {{ .Values | get "apps.new-app" false }}`
3. Add `new-app: true` to the relevant `environments/*.yaml`
4. Create `values_overwrite/values_new-app.yaml` (or run `generate-values.sh`)
